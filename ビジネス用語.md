# ビジネス用語まとめ
## ①Google APIの「上限（Quota）」とは
👉 APIを 一定回数までしか使えない制限 のこと

**主な制限例**
- 1日〇回まで
- 1分〇回まで
- 1ヶ月〇回まで
- Google Maps API：月 $200 までは無料、超えると課金

**上限を設定しないリスク**
- 課金爆発
- APIキーが漏れる → 第三者が大量リクエスト → 高額請求
- DDoS的悪
- botで大量アクセス → 課金発生

**推奨されるセキュリティ設定**
- APIキー制限（ドメイン制限・IP制限）
- APIごとの利用制限
- Quota（上限）設定

👉 結論：**上限設定は課金事故防止のために必須**

## ②WordPress（ワードプレス）とは
👉 世界で最も使われている CMS（コンテンツ管理システム）

**できること**
- ブログ
- 企業サイト
- ECサイト
- メディアサイト

**特徴**
- 世界のWebサイトの約40%がWordPress

**技術：PHP / MySQL**

- テーマ・プラグインで拡張可能
- Laravelエンジニアでも案件で触る可能性が高い

## ③ピボット（Pivot）
👉 事業やサービスの方向転換 のこと

例：
**個人向け → 法人向け**

アプリ → SaaS

**有名例**：Instagram
位置情報アプリ → 写真SNSへ変更

## ④SaaS（サース）
👉 Software as a Service
👉 インストール不要で、ブラウザで使うクラウドソフト

**代表例**
- Slack
- Notion
- Google Docs
- Salesforce

**会社の業務ツールの多くはSaaS**

## ⑤MVP（Minimum Viable Product）
👉 最小限の機能だけ作った製品

**例**
- SNSを作る
→ 最初は「投稿機能」だけ
→ いいね・フォローは後で追加

## ⑥スケール（Scale）
👉 サービスを 大きく成長させること

例：ユーザー100人 → 10万人

## ⑦KPI（Key Performance Indicator）
👉 重要な数値目標

例：登録数 / 売上 / アクセス数

## ⑧リファクタリング
👉 動作を変えずにコードをきれいにすること

Laravel開発では頻出

## ⑨技術的負債（Technical Debt）
👉 急いで作った結果、後で修正コストが増える汚いコードが残ること

## ⑩PoC（Proof of Concept）
👉 実験的なプロトタイプ

例：AIで画像認識できるか？ → 簡易試作

##  ⑪グロース（Growth）
👉 サービスを 成長させる活動。

例：ユーザー増加、売上増加

## ⑫デプロイ（Deploy）
👉 コードを本番環境に反映すること

例：git push → 本番更新

## ⑬BtoB / BtoC
👉 BtoB：企業向け
👉 BtoC：一般ユーザー向け

## ⑭アセット（Asset）
👉 会社の 資産

例：コード / データ / 顧客情報 / ドメイン / ブランド

## ⑮レガシー（Legacy）
👉 古くて保守が大変なシステム

例：古いPHP、古いDB、仕様不明のシステム

## ⑯ドメイン（Domain）
意味は2つ
👉 Webドメイン：example.com
👉 ビジネスドメイン：業界領域（医療 / 金融 / 教育など）

## ⑰バーティカルSaaS
👉 特定業界向けのSaaS

**種類**
- 横断型：Slack、Google Workspace
- バーティカル型：美容院向け、歯医者向け、不動産向け

## ⑱PM（プロダクトマネージャー）
**役割**
- 何を作るか決める
- 優先順位を決める
- ビジネス判断を行う
- エンジニアと最も関わる職種

## ⑲KGI（Key Goal Indicator）
👉 KPIの上位目標

**例**
- KGI → 売上1億
- KPI → 登録数 / CVR

## ⑳CVR（Conversion Rate）
👉 コンバージョン率

例：100人訪問 → 10人登録 → CVR 10%

## ㉑LTV（Life Time Value）
👉 顧客が生涯で支払う金額

例：月1000円 × 12ヶ月 = LTV 12000円

## ㉒チャーン（Churn）
👉 ユーザーが離れること（解約）

チャーン率＝解約率

## ㉓ARR（Annual Recurring Revenue）
👉 年間サブスク売上
