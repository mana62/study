# クラスのアクセス修飾子
＝PHPのアクセス修飾子（3種類）

```php
class Pokemon {
    // public - どこからでもアクセス可能
    public $name = "ピカチュウ";
    
    // protected - クラス内部 + 継承先でアクセス可能
    protected $level = 1;
    
    // private - クラス内部のみアクセス可能
    private $hp = 100;
    
    public function heal() {
        $this->hp += 10;  // OK: クラス内部
    }
}

class Pikachu extends Pokemon {
    public function showLevel() {
        echo $this->level;  // OK: protectedは継承先で使える
        echo $this->hp;     // エラー！privateは継承先でも使えない
    }
}

$pikachu = new Pikachu();
echo $pikachu->name;   // OK: public
echo $pikachu->level;  // エラー！protected
echo $pikachu->hp;     // エラー！private
```

## まとめ
| アクセス修飾子 | クラス内部 | 継承先 | 外部 |
|----------------|-----------|--------|------|
| public         | ✅        | ✅     | ✅   |
| protected      | ✅        | ✅     | ❌   |
| private        | ✅        | ❌     | ❌   |


## アクセス修飾子はクラスのみに使えるもの？？？
| 言語        | クラス専用？ | 備考 |
|-------------|--------------|------|
| JavaScript  | ✅ はい      | `#` はクラスのプライベートのみ |
| PHP         | ✅ はい      | クラスメンバーにのみ使用可能 |
| Java        | ❌ いいえ    | クラス自体にもアクセス修飾子を付けられる |
| TypeScript  | ❌ いいえ    | 引数、プロパティ、クラスなど幅広く使える |
| C#          | ❌ いいえ    | ほぼすべてのメンバーに使用可能 |
| Kotlin      | ❌ いいえ    | トップレベル関数にも付けられる |
| Python      | ⚠️ 無し     | アクセス修飾子は存在せず、慣習（`_` や `__`）のみ |
