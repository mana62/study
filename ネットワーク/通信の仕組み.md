# インターネット通信の仕組み
**プロトコル** 👉 通信のルール
**バイナリデータ** 👉 ２進数のことで、PCがやり取りする時の値
**TCP** 👉 「確実に届ける」ことを重視する通信方法
**UDP** 👉 「速さ」を重視する通信方法
**ASCII（アスキー）** 👉 文字をコンピューターが分かる数字に変換するルール（0か1）
**通信のやり取り** 👉 ①IPアドレス ②ポート番号 を使用する
**DNS** 👉 ドメイン名をIPアドレスに変換する仕組み

## ① クライアント・サーバーとは？

```
┌────────────┐            ┌────────────┐
│ クライアント │  リクエスト → │   サーバー   │
│ (ブラウザ)   │            │ (Webサーバー) │
└────────────┘ ← レスポンス  └────────────┘
```

### クライアント

* リクエスト（お願い）を送る側
* 例：ブラウザ、スマホアプリ

### サーバー

* リクエストを受け取り、処理して返す側

#### リクエスト
- メソッド（GETやPOST）
- パス
- ヘッダー（key & value）
- ボディ（フォーム）

#### レスポンス
- ステータスコード
- ヘッダー（key & value）
- ボディ（HTML & 画像 など）

---

## ② 通信の流れ（全体像）

```
[自分]
   ↓
[ブラウザ]
   ↓ ①HTTPリクエスト
──────────────────────────
        インターネット
──────────────────────────
   ↓
[Webサーバー]
   ↓ ②HTTPレスポンス
──────────────────────────
   ↓
[ブラウザ]
   ↓
[画面表示]
```

---

## ③ プロトコルの階層

```
┌──────────────────────────┐
│ アプリケーション層       │  ← HTTP / HTTPS
├──────────────────────────┤
│ トランスポート層         │  ← TCP / UDP
├──────────────────────────┤
│ インターネット層         │  ← IP
├──────────────────────────┤
│ ネットワーク層           │  ← LAN / Wi-Fi
└──────────────────────────┘
```

---

## HTTP / HTTPS

* データのやり取りのルール
* 「何をください」「これを返します」

---

## TCP と UDP

### TCP（確実だが遅め）

```
送信 → 確認応答 ←
送信 → 確認応答 ←
送信 → 確認応答 ←
```

✔ データ保証あり
✔ 順番保証あり
✔ Web・メール向き

---

### UDP（高速だが保証なし）

```
送信 → →
送信 → →
送信 → →
（確認しない）
```

✔ とにかく速い
✔ ゲーム・動画向き

---

## ④ 実際の通信の内部イメージ

```
HTTPデータ
   ↓
TCPが細かく分割（パケット化）
   ↓
IPが宛先アドレスをつける
   ↓
ネットワークへ送信
```

サーバー側では逆順で組み立てる

---

## ⑤ URLの構造と役割

例：

```
https://example.com:443/blog/index.html?id=10#top
```

分解すると：

```
[スキーム]://[ホスト名]:[ポート]/[パス]?[クエリ]#[フラグメント]
```

---

## ① スキーム

```
https
```

* 通信方式
* http / https / ftp など
* URL外では、**プロトコル** と呼ばれる

---

## ② ホスト名（ドメイン）

```
example.com
```

* 接続先のサーバー名
* DNSでIPアドレスに変換される

---

## ③ ポート番号

```
:443
```

* サーバー内の入り口番号
* 80 → HTTP
* 443 → HTTPS

---

## ④ パス

```
/blog/index.html
```

* サーバー内のファイルやページの場所

---

## ⑤ クエリパラメータ

```
?id=10
```

* 追加情報
* サーバーに渡すデータ
* 「idが10のデータをください」など

---

## ⑥ フラグメント

```
#top
```

* ページ内ジャンプ用
* サーバーには送られない

---

# 全体図

```
URL入力
   ↓
DNSでIP取得
   ↓
TCP接続確立（3-way handshake）
   ↓
HTTPリクエスト送信
   ↓
サーバー処理
   ↓
HTTPレスポンス
   ↓
TCP切断
   ↓
画面表示
```

---

# まとめ

* クライアント → リクエスト送信
* サーバー → レスポンス返す
* HTTP → 会話の内容
* TCP/UDP → 運び方
* IP → 住所指定
* URL → 住所＋通信方法＋追加情報
* ASCII → 文字をコンピューターが分かる数字に変換するルール（0か1）
