# 中間テーブル

👉 中間テーブルを使う **多対多（belongsToMany）** の場合は、
基本的に **attach / detach / sync を使って紐付けを操作する** という仕組みになっている


```php
public function tags()
{
    return $this->belongsToMany(Tag::class);
}
```
＝ モデルにメソッドがあることで、Laravel はこう理解する：
- 「Article と Tag は多対多だから、中間テーブルを使って紐付けるんだな」
- そして、**紐付け操作をするためのメソッドが attach / detach / sync**

## attach を使うと何が起きる？
```php
$article->tags()->attach([1, 3, 5]);
```
これは内部的にこういう SQL を実行：

```bash
INSERT INTO article_tag (article_id, tag_id) VALUES (10, 1);
INSERT INTO article_tag (article_id, tag_id) VALUES (10, 3);
INSERT INTO article_tag (article_id, tag_id) VALUES (10, 5);
```
つまり：
- 記事ID = 10
- タグID = 1, 3, 5
を中間テーブルに保存する

## なぜ tags() に () が必要なのか？
- tags（プロパティ）
→ **実際のタグ一覧（コレクション）**

- tags()（メソッド）
→ **リレーションを操作するための「関係オブジェクト」**

👉 attach は リレーションに対して使うメソッド なので、**tags() のように () をつけてメソッドとして呼び出す必要がある**

## attach 以外に何がある？
＝ 多対多では以下の3つが基本

### 1. attach（追加）
```php
$article->tags()->attach([1, 2]);
```

### 2. detach（削除）
```php
$article->tags()->detach([1]);
```

### 3. sync（更新）
```php
$article->tags()->sync([2, 3]);
```
＝ sync は「今のタグを全部これに置き換える」という動き

## まとめ
- **多対多では attach / detach / sync を使う**
- attach は中間テーブルにレコードを追加する
- tags() はリレーションを返すメソッドなので () が必要
- tags は実際のタグ一覧なので attach は使えない

## with() ではなく load() を使う理由
- with() → クエリビルダ用（複数件取得するとき）
- load() → すでに取得済みのモデルに対して使う
- show() は 1件のモデルが渡されているので load() が正しい
