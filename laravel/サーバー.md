# Laravel Cloud / Laravel Forge / Laravel Nightwatch について

## Laravel Cloud（2025年2月リリース）
- Laravel公式が提供するフルマネージドなクラウドデプロイサービス
- Laravelアプリをすぐに本番環境へデプロイできる

### 特徴
- サーバー構築不要：インフラの準備はすべて自動
- PHP / Nginx 設定不要：Laravel に最適な設定が自動で適用
- SSL 証明書の自動発行：Let's Encrypt 対応で HTTPS もすぐに
- GitHub 連携でデプロイ：プッシュするだけで自動デプロイ
- スケーラビリティ対応：アクセス増加時も自動でスケール
- 環境変数の GUI 管理：.env の編集もブラウザから簡単に
- データベースやキャッシュの管理も GUI で可能

### メリット
- インフラ不要で即デプロイ可能
- GitHub 連携で自動デプロイ
- SSL やスケーリングも自動
- Nightwatch と連携可能

### デメリット
- 自由度が低い（細かい設定ができない）
- 自前サーバーが使えない
- Forge よりコストが高くなる可能性

### 注意点
- Laravel Cloud 専用の構成になるため、他のフレームワークや用途には不向き
- Nightwatch などの機能は Cloud 環境に依存

---
## Laravel Forge（2014年リリース）
- 自分で契約したサーバーをLaravel用に自動構成してくれるサーバー管理ツール
- Laravel Cloud より自由度が高く、インフラのカスタマイズも可能

### 特徴
- 主要クラウド/VPSと連携：AWS, DigitalOcean, Linode などに対応
- PHP / Nginx / MySQL 自動インストール
- Laravelアプリの初期設定を自動化
- GitHub / GitLab連携でデプロイ自動化
- SSL証明書の発行・更新（Let's Encrypt）
- キュー・スケジューラー（cron）の管理
- ファイアウォールや SSH キーの管理も GUI で簡単
- 複数サイトのホスティングに対応

### メリット
- 自分でサーバーを選べる
- 高い自由度とカスタマイズ性
- GUI でサーバー管理が簡単
- コストを抑えやすい

### デメリット
- サーバーの基本知識が必要
- セキュリティやスケーリングは自分で対応
- Nightwatch などの一部機能が使えない

### 注意点
- サーバーの障害対応や監視は自前で用意する必要あり
- Laravel Cloud との連携は現時点では不可（将来的に変わる可能性あり）

---
## Laravel Nightwatch（2025年2月リリース）
- Laravel Cloudと連携して使える本番環境の監視・可視化ツール
- アプリの状態をリアルタイムで監視し、問題をすばやく発見・対応できる

### 特徴
- 例外・エラーのリアルタイム検知
- スタックトレース付きの例外ログ表示
- 遅いSQLクエリや処理の特定
- ログの一元管理と検索機能
- Slack / メール通知対応
- パフォーマンスメトリクスの可視化（CPU, メモリ, レスポンスタイムなど）
- デプロイ後の自動チェック機能

### メリット
- 本番環境のエラーや遅延をリアルタイムで検知
- Slack やメール通知で即対応可能
- Cloud と連携して自動で導入される

### デメリット
- Laravel Cloud 専用（単体利用不可）
- 他の監視ツールに比べてカスタマイズ性は低め

### 注意点
- Laravel Cloud でのデプロイが前提条件
- 他の監視ツールとの併用は要検討（重複通知など）
