# MCP
- 例えば、chatGPTに、グーグルカレンダーに⚪︎⚪︎を登録してと伝えても、できない。けど、MCPサーバーを挟めば、chatGPT → MCPサーバー (GoogleカレンダーのAPIに伝えられる) → Googleカレンダー
- MCP（Model Context Protocol）＝ AIと外の世界 (天気・スラック・他のAI) をつなぐための共通の橋
- AI（GPTやClaude）は「頭脳」
- MCPサーバーは「手足＋通訳」


---

## 用語

| 役割                      | たとえ | 実際の仕組み                |
| ----------------------- | --- | --------------------- |
| **AIモデル（GPTやGemini）**   | 頭脳  | 言葉を理解して考える            |
| **MCPサーバー**             | 手と口 | 「外の世界（APIやPC操作）」とつなぐ橋 |
| **外部API（例：Gemini API）** | 道具  | 情報を取ったり、処理を代行したりする    |
| **MCPプロトコル**                       | 言語・ルール  | AIとサーバーがやり取りする形式            |
| **MCPサーバー**                        | 実体（アプリ） | AIからのMCPリクエストを受けて外部APIに接続する |
| **AIエージェント（CursorやClaude Codeなど）** | クライアント  | MCPプロトコルを話せる“話し手”           |




| 用語                              | 役割                                          | 例                                 |
| ------------------------------- | ------------------------------------------- | --------------------------------- |
| **AIモデル**                       | 質問応答・推論を行う「脳」                               | GPT-5、Gemini 1.5、Claude 3など       |
| **AIエージェント**                    | モデルに加えて、ツールやAPIを使って自律的に行動する存在               | OpenAI ChatGPTの「エージェント」モードなど      |
| **MCP（Model Context Protocol）** | モデルと外部システム（DB、API、CLIなど）を安全に接続するための新しいプロトコル | Anthropic（Claude）発祥。GPTでも対応が進んでいる |
| **MCPサーバー**                     | MCP対応のAIが利用できるように、外部ツール・APIを登録する中間サーバー      | 例：ファイル操作サーバー、DBサーバー、HTTPコールサーバーなど |




```bash
[私]
   ↓
[AIエージェント（GPT-5など）]
   ↓
[MCPサーバー] ← ← ← ここが「手足」
   ↓
[外部APIやサービス（例：天気API、Gemini API、DB など）]
```

```bash
[Laravel Boost]
   ↓（接続設定）
[Cursor or Claude Code] ←─── (AIエージェント・MCP対応)
   ↓（MCPプロトコル）
[MCPサーバー（あなたが立てる）]
   ├─ Gemini API（無料版OK）
   ├─ 天気API
   ├─ Slack API
   └─ その他のサービス
```

---

## 「MCPサーバー」がやってること

| 機能          | 具体的な動き                          |
| ----------- | ------------------------------- |
| 通信の翻訳    | AIの命令を外部APIが理解できる形式に変換          |
| セキュリティ制御 | どのAPIを使えるか、どんな操作が許されるか制御        |
| データのやり取り | APIからの結果をAIにわかる形式で返す            |
| 拡張可能     | 天気APIでもSlackでもGeminiでも、自由に追加できる |

- GPT-5 や Claude みたいなMCP対応AIは、「MCPサーバーを通して、好きなAPIを組み合わせられる」
- Gemini API のような普通のAPIは、「MCPサーバーの中で使われる（＝道具）」という立ち位置



---

## 何ができるのか
- AIがカレンダーやメール、データベースと直接やりとりできる
- ツールごとに連携コードを書かなくていい（開発コストが減る）
- AIが「文脈」を理解しながら、外部ツールを使えるようになる


---

## MCPのしくみ

| 要素             | 役割                                   |
| ---------------- | -------------------------------------- |
| MCPクライアント     | AIが使う「操作の窓口」                         |
| MCPサーバー       | 外部ツールとつなぐ「橋渡し役」                   |
| MCPプロトコル     | クライアントとサーバーが話すための「共通言語」       |

---

## どんなツールとつながるのか
- Google Workspace（カレンダー・ドキュメント）
- SQLデータベース
- REST API / GraphQL
- メッセージキュー（Kafka / RabbitMQ）
- ファイルストレージ（Dropboxなど）

---

## メリット

| メリット           | 説明                                                         |
| ------------------ | ------------------------------------------------------------ |
| 開発がラクになる     | ツールごとの専用コードが不要になる                                     |
| 拡張性が高い         | 新しいツールもMCPに対応していればすぐ連携できる                        |
| AIの精度が上がる     | 外部データを活用して、より正確な回答ができるようになる                     |
| 業務効率がアップ     | AIが自動でツールを操作してくれるから、作業がスムーズになる                     |
