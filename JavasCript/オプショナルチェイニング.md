# オプショナルチェイイニング
＝ 存在しないかもしれない値に安全にアクセスするための書き方
＝ ?をつけることで左側の値がnullであればundefindを返し、値があれば値を返す

## 使える場面
① オブジェクトのプロパティ
```js
user?.profile?.name
```

② 配列の要素
```js
items?.[0]?.title
```

③ 関数呼び出し
```js
callback?.()
```

④ メソッド呼び出し
```js
user?.getName?.()
```

## 注意
❌ 左側が 必ず null/undefined 以外 になることが前提の場所では使えない
```js
// 例：代入の左辺とかは NG
// ❌ これは文法エラー
user?.name = "Taro"
```

❌ オプショナルチェイニングを「書きすぎる」と逆に読みにくくなる
```js
// NG
a?.b?.c?.d?.e?.f
```
