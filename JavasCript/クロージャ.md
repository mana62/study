# クロージャ
= 関数とその変数が宣言されたレキシカル環境

## クロージャができること
①関数の中で関数を定義できる
②関数を変数に入れて持ち運べる
③親の関数の変数を使える。
 ※ 関数が２つあった場合、一方の関数内から他方の関すの変数を使うことはできない
 ※ 親子関係にある関数のみ、だが親 → 子 ❌ 子 → 親 ⭕️（何個でもOK **スコープチェイン**と呼ぶ）
④JSの関数の親とは産みの親のこと
  - 1. ダイナミックスコープ : 関数が実行された時の親
  - 2. レキシカルスコープ : 関数が定義された、作られた時の親

→ 「レキシカルスコープ : 関数が定義された、作られた時の親」この親が持ってる変数などをレキシカル環境で保持している


```js
let a = 1;

function outer() {
  let b = 2;

  function inner() {
    let c = 3;
    console.log(a, b, c);
  }

  inner();
}
outer();
```
・inner は outer の中で定義された
・outer はグローバルの中で定義された
・だから inner の親は outer
・outer の親は global
= この“親関係”を決めるルールがレキシカルスコープ


### ✔ レキシカルスコープ
「この関数はどこの親に属しているか」
＝ 関数が“どこで書かれたか”で決まる

### ✔ レキシカル環境
「そのスコープの中に実際に入っている変数のセット」
＝ 実行コンテキストの中にある“変数の倉庫”
