# 実行コンテキストとスタック

実行コンテキスト : 関数ごとの作業部屋
スタック : その作業部屋を積み重ねられる棚

✅ 関数を呼ぶたびに作業部屋が作られ、棚に積まれる
✅ 終わったら棚から取り除かれる
✅ メモリ内（ブラウザ）で行っていること
✅ **ガーベージコレクション** : メモリ内（ブラウザ）で自動で、不要なデータを削除すること
✅ **ブラウザの最適化（ブラウザの種類による）** : メモリ内のV8エンジンが使われていない値などを整理して最適化する

### 実行コンテキストの中身

```bash
┌──────────────────────────────┐
│        実行コンテキスト (Execution Context)        │
├──────────────────────────────┤
│ ① レキシカル環境 (Lexical Environment)            │
│    ├─ 環境レコード (Environment Record)            │
│    │     ├─ 変数 (let / const)                    │
│    │     ├─ 関数宣言                              │
│    │     └─ 引数                                  │
│    └─ 外部環境参照 (Outer Environment Reference)  │
│          → 親スコープへのリンク（スコープチェーン）│
├──────────────────────────────┤
│ ② 変数環境 (Variable Environment)                 │
│    ├─ var の変数                                  │
│    └─ 関数宣言（ここに入ることもある）            │
├──────────────────────────────┤
│ ③ this バインディング (this Binding)             │
│    → この実行コンテキストでの this の値           │
└──────────────────────────────┘
```
