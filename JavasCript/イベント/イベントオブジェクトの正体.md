# イベントオブジェクトの正体
＝**イベントオブジェクトは1個だけ**
👉 キャプチャ → ターゲット → バブリング の全工程で同じイベントオブジェクトが使い回されている

そのため
```bash
captureEvent === targetEvent === bubblingEvent
```
は `true` になる = **全部同一のイベントオブジェクト**

## 重要プロパティ① currentTarget
```js
event.currentTarget
```
👉 「今まさに実行中のイベントリスナーが付いている要素」

```bash
window → document → body → input
```
| 実行場所     | currentTarget |
| -------- | ------------- |
| window   | window        |
| document | document      |
| body     | body          |
| input    | input         |

### this と同じ意味
通常の function で書くと：
```js
this === event.currentTarget
```
になる
※ アロー関数だと this は変わらないので注意

## 重要プロパティ② eventPhase
```js
event.eventPhase
```
👉 今どのフェーズかを数字で教えてくれる

| 数字 | 意味        |
| -- | --------- |
| 1  | キャプチャフェーズ |
| 2  | ターゲットフェーズ |
| 3  | バブリングフェーズ |
| 0  | 全処理終了後    |


## なぜ同じオブジェクトを使い回すのか
👉 途中で値を書き換えて、後続の処理に影響を与えられるようにするため

```js
event.stopPropagation();
```
＝これは 同じイベントオブジェクトを共有しているから止まる

## まとめ
イベントの正体：
- イベント発生
- 1個のイベントオブジェクトが生成
- キャプチャ → ターゲット → バブリング へと
- 同じオブジェクトが順番に回される
