# 非同期処理
＝JSは **シングルスレッド**（シングルスレッド:一度に1つの処理しか実行できない）
＝setTimeout のような非同期APIは、
**JavaScriptエンジン単体ではなく**、**実行環境（ブラウザやNode.js）が提供している仕組み** によって動いている

### 詳しくいうと
JavaScript自体は：
- **シングルスレッド**
- **非同期処理の仕組みを言語仕様としては持っていない**

つまり：
👉 **非同期に見える処理は、実行環境が裏でやってくれている**

## 例えばsetTimeout の中身
**ブラウザの場合**
```bash
JavaScript
   ↓
Web APIs（ブラウザの機能群：C++などで実装）
   ↓
タイマー管理
   ↓
イベントキュー
   ↓
JavaScript実行
```

**Node.js の場合**
```bash
JavaScript
   ↓
libuv（C言語で書かれた非同期I/Oライブラリ）
   ↓
イベントループ
   ↓
JavaScript実行
```

## まとめ
非同期処理は、JavaScriptがやっているのではなく、ブラウザ（やNode.js）が裏でやってくれている

**非同期処理 = ブラウザが裏で実行**
**JS = 指示と結果処理だけ**
