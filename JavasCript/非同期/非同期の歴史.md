# 非同期の歴史
＝前提：非同期とは、ある処理を待たずに次の処理に進むこと、時間がかかる処理を待たない

**昔：非同期＝コールバック関数**
→ でも、処理が増えると「コールバック地獄（callback hell）」

**Promiseの登場（ES6）**
→ 「非同期をチェーンでつなげよう！」っていう新しい書き方が登場

さらに **async/await（ES2017）**
→ 「非同期を同期っぽく書ける」ってことで、読みやすさが爆上がり

---

⚫︎昔は非同期はコールバック関数で書いていた → ①コールバック地獄になった。②非同期は`try catch`が使えない
**コールバック地獄：ネストが深くなりすぎること**

⚫︎やりやすくするために、**promise**が生まれた
⚫︎更にやりやすくするために、**async/await**が生まれた
