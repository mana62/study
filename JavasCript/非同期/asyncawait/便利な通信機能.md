# 便利な通信機能

## ① fetchは途中でキャンセルできる
👉 **AbortController** を使う

例👇
```js
const controller = new AbortController();

fetch("https://api.example.com", {
  signal: controller.signal
});

controller.abort(); // fetch中断
```
**用途**
- ページ移動した
- 検索をキャンセル
- 無駄な通信を止める
👉 fetchを途中で止める機能

## ② 昔は fetch がなかった
昔のHTTP通信は **XMLHttpRequest** を使っていた

例
```js
const xhr = new XMLHttpRequest();
xhr.open("GET", "/api");
xhr.send();
```

**特徴**
| 項目      | fetch | XMLHttpRequest |
| ------- | ----- | -------------- |
| 新しさ     | 新しい   | 古い             |
| Promise | 使える   | 使えない           |
| 使いやすさ   | 簡単    | 面倒             |

👉 今は基本 fetch を使う

## ③ サーバー→クライアント通信の方法
普通のHTTP **Client → Server** しかできない
つまり **サーバー → ブラウザ** はできない

👉 だから特別な方法がある

### 方法① Server-Sent Events（SSE）
- サーバーが **一方的にデータを送り続ける**

図👇
```bash
Server
   │
   │ データ
   │ データ
   │ データ
   ▼
Browser
```
```js
const eventSource = new EventSource("/events");

eventSource.onmessage = (e) => {
  console.log(e.data);
};
```
**特徴**
| 特徴   | 内容          |
| ---- | ----------- |
| 方向   | サーバー → ブラウザ |
| データ  | 文字列         |
| 接続切れ | 自動再接続       |

**用途**
- 通知
- 株価
- ライブ更新

### 方法② WebSocket
これは **リアルタイム双方向通信**

図👇
```bash
Browser ⇄ Server
```

```js
const ws = new WebSocket("wss://example.com");

ws.onmessage = (e) => {
  console.log(e.data);
};

ws.send("hello");
```

**特徴**
| 特徴     | 内容        |
| ------ | --------- |
| 通信     | 双方向       |
| データ    | 文字 + バイナリ |
| リアルタイム | ◎         |

**用途**
- チャット
- ゲーム
- ライブ配信
- コラボ編集

### SSE vs WebSocket
| 項目    | SSE         | WebSocket |
| ----- | ----------- | --------- |
| 方向    | サーバー→クライアント | 双方向       |
| プロトコル | HTTP        | 専用        |
| データ   | 文字          | 文字＋バイナリ   |
| 実装    | 簡単          | 少し難しい     |

## まとめ
- fetch追加知識 ＝ **AbortController → fetch中断**

- 古い通信：`XMLHttpRequest` （今はほぼ使わない）

- **サーバーからデータ送る方法**
1️⃣ Server-Sent Events
`Server → Browser`
2️⃣ WebSocket
`Browser ⇄ Server`

**使い分け**
| 用途     | 技術        |
| ------ | --------- |
| 普通のAPI | fetch     |
| 通知     | SSE       |
| リアルタイム | WebSocket |
