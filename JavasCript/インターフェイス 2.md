# インターフェイス
＝何かと何かが **やり取りするための窓口・約束事**
＝「どうやって話す？」「どんな形式で渡す？」という ルール が必要 → **そのルールが インターフェイス**

## プログラミングでのインターフェイス
### ① 外部に公開する 使い方の約束
```js
element.addEventListener("click", handler);
```
どんな関数を渡すか
どんなイベント名を使うか
＝これが インターフェイス

### ② TypeScript の interface（型の約束）
```js
interface User {
  name: string;
  age: number;
}
```
これは 「この形でデータを渡してね」 という約束

### ③ ライブラリやクラスの 外から触れる部分
```js
class Car {
  start() {}
  stop() {}
}
```
start() と stop() が外部とのインターフェイス

## まとめ
どうやって使うか
何を渡すか
何が返ってくるか
どんな形式でやり取りするか
→ こういった **「やり取りのルール」 を定義したもの**
