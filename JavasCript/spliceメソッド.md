# spliceとtoSpliced

## splice
＝ **配列**に対して使う
＝ 配列の好きな場所の**値を消したり、値を追加したり**できる
＝ 引数は３つ取れる

```js
array.splice(開始位置, 削除する数, 追加する値1, 追加する値2, ...)
```

```js
// 削除例
const arr = [1, 2, 3, 4];
arr.splice(1, 2);

console.log(arr); // [1, 4]
```

```js
// 追加例
const arr = [1, 2, 3];
arr.splice(1, 0, "A"); // 0は何も削除しない

console.log(arr); // [1, "A", 2, 3]
```

## toSpliced

- splice
→ 元の配列を直接いじる（破壊する）

- toSpliced
→ 元の配列はそのまま、新しい配列を返す（非破壊）

＝やってること（削除・追加・置換）は同じだけど、「元の配列を壊すかどうか」だけが違う



## まとめ

| メソッド     | 配列を変更する？       | 返り値         |
|--------------|-------------------------|----------------|
| splice       | 変更する（破壊的）     | 削除した要素   |
| toSpliced    | 変更しない（非破壊）   | 新しい配列     |
