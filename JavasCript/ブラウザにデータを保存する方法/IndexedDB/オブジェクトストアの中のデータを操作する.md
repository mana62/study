# オブジェクトストアの中のデータを操作する方法

## IndexedDBにデータを入れる流れ
データを保存するには 下記順番
- ① Transactionを作成
- ② ObjectStoreを取得
- ③ putでデータを追加

コードイメージ👇
```js
const tx = db.transaction("books", "readwrite") // 第二引数に操作モードを
const store = tx.objectStore("books")

store.put("JavaScriptガイド", 0) // 必ず第二引数にkeyが必要
```

## Transaction（トランザクション）とは
- データ操作をするための作業モード

```js
db.transaction("データベース名", "操作モード");
// 例
db.transaction("books", "readwrite");
```

| モード       | 意味   |
| --------- | ---- |
| readonly  | 読むだけ |
| readwrite | 読み書き |
**※書き込みするなら readwrite**

## データ追加（put）
👉 必ず第二引数にキーを入れる（複数のvalueがあっても識別できるようにするため）
```js
store.put("値", "キー");
// 例
store.put("JavaScriptガイド", 0)
```
意味：
- キー: 0
- 値: "JavaScriptガイド"

## IndexedDBの構造
**IndexedDBは 3階層**
```bash
Database
   ↓
Object Store
   ↓
key → value
```
例：
```bash
shop（Database）
 └ books （Object Store）
     ├ 0 → "JavaScriptガイド" （key → value）
     └ 1 → "Python" （key → value）
```

## データ取得
```js
store.get(0)
```
👉 ただし **非同期処理** なので

```js
request.onsuccess = () => {
  console.log(request.result)
}
```

## データ削除
**1つ削除**
```js
store.delete(0)
```
**全部削除**
```js
store.clear()
```

## ⚠️ IndexedDBの重要ポイント
**非同期**
- 全部 **非同期処理**
```bash
open
get
put
delete
```
**実行順序**
- 呼び出し順に処理される
```bash
put
put
get
```
なら必ず `put → put → get` の順で実行される

## 保存できるデータ
**保存できるもの**
- 文字列
- 配列
- オブジェクト
- Map
- Set
- Blob

**保存できないもの**
- 関数
- DOM
- WebAPIオブジェクト

## まとめ
**IndexedDBでデータ保存**
① DBを開く
② Transaction開始
③ ObjectStore取得
④ put()で保存
⑤ get()で取得

**シンプルなコード例**
```js
const tx = db.transaction("books", "readwrite")
const store = tx.objectStore("books")

store.put("JavaScriptガイド", 0)

const req = store.get(0)

req.onsuccess = () => {
  console.log(req.result) // result は request オブジェクトが持っているプロパティ
}
```
**ブラウザの内部処理**
```bash
store.get(0) 呼び出し
      ↓
Requestオブジェクト作成
      ↓
DBからデータ取得開始（非同期）
      ↓
データ取得完了
      ↓
request.result にデータ保存
      ↓
successイベント発生
      ↓
onsuccess の関数が実行される
```
