# オブジェクトのプロパティの種類
⚫︎データプロパティ（普通のプロパティ）
⚫︎アクセスプロパティ **（getter / setter を使うプロパティ）**
アクセスプロパティは、値を直接持たず、**関数を通して値を読み書き**する仕組み

## データプロパティ（普通のやつ）
```js
const obj = {
  name: "Alice"
};

console.log(obj.name); // "Alice"
```

## アクセスプロパティ（getter / setter）
```js
const user = {
  _age: 20,

  get age() {
    return this._age;
  },

  set age(value) {
    if (value >= 0) {
      this._age = value;
    }
  }
};

console.log(user.age); // 20（関数が呼ばれている）
user.age = 25;         // setter が呼ばれる
```
ここで age は アクセスプロパティ
- get age() → 読み取り時に呼ばれる関数
- set age() → 書き込み時に呼ばれる関数

## アクセスプロパティの特徴
| 種類 | 説明 |
|------|------------------------------|
| 値を直接持たない | 値は内部の変数（例：_age）に保存する |
| 読み書き時に関数が動く | get/set が自動で呼ばれる |
| バリデーションができる | setter で値のチェックが可能 |
| 計算結果を返せる | getter で動的に値を作れる |


## 何が便利なのか
⚫︎値のチェックができる
```js
set age(value) {
  if (value < 0) throw new Error("年齢は0以上");
}
```

⚫︎計算結果を返せる
```js
get fullName() {
  return `${this.first} ${this.last}`;
}
```

⚫︎外からは普通のプロパティに見える
```js
console.log(user.age); // 関数が動いてるのに普通のプロパティみたいに使える
```

## まとめ
- アクセスプロパティ = **getter / setter を使った特別なプロパティ**
- 値を直接持たず、関数を通して読み書きする
- バリデーションや計算に使える
- 外からは普通のプロパティのように見える
