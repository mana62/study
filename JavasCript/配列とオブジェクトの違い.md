# 配列とオブジェクトの違い
- **オブジェクト** キーとvalueで管理したい時
- **クラス** 一個のオブジェクトを何個も作りたい時
- **配列** 単純に同じ分類や種類をまとめたい時
- **配列＆オブジェクト** 同じ種類の値をキーとバリューでまとめたい時



## 配列
順序が重要な場合や、同じ種類のデータをまとめるとき

```js
// リスト、順序のあるデータ
const fruits = ['りんご', 'バナナ', 'みかん'];
const scores = [85, 92, 78, 95];

// 順番通りにアクセス
fruits[0]; // 'りんご'
fruits.forEach(fruit => console.log(fruit)); // 順番に処理
```
### 配列を使う時
- ループで処理したいデータの集まり
- ランキング
- 履歴データ など

## オブジェクト
キーを使ってプロパティを管理したい時
```js
// 1つのものの属性を表現
const user = {
  name: '田中太郎',
  age: 25,
  email: 'tanaka@example.com'
};

// 名前でアクセス
user.name; // '田中太郎'
user['email']; // 'tanaka@example.com'
```

### オブジェクトを使う時
- ユーザー情報
- 設定データ
- キーと値のペア

## 配列とオブジェクトを組み合わせで使う時
同じ種類の値をキーとバリューでまとめたい時

```js
// ユーザーのリスト（配列の中にオブジェクト）
const users = [
  { name: '田中', age: 25 },
  { name: '佐藤', age: 30 },
  { name: '鈴木', age: 28 }
];

// ユーザーが持つ趣味のリスト（オブジェクトの中に配列）
const userProfile = {
  name: '田中',
  hobbies: ['読書', '映画', 'ゲーム']
};
```
